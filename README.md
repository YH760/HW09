# 숫자야구 멀티플레이 (ChatX)

언리얼 엔진 기반 **멀티플레이 숫자야구** 게임. 서버가 정답을 생성하고 판정하는 **서버 권위(Server-Authoritative)** 구조로, 채팅으로 숫자를 입력해 스트라이크/볼을 맞춘다.

## 게임 규칙

- 서버가 1~9 사이 **중복 없는 3자리 숫자**를 정답으로 생성
- 플레이어는 채팅으로 3자리 숫자를 입력해 정답을 추측
- 판정 결과
  - **S(Strike)**: 숫자와 자리가 모두 일치
  - **B(Ball)**: 숫자는 있지만 자리가 다름
  - **OUT**: 일치하는 숫자가 하나도 없음
- 각 플레이어는 **3번의 기회**를 가지며, 먼저 3S를 맞추면 승리
- 전원이 기회를 소진하면 무승부

## 구현 기능

### 필수 기능
- 멀티플레이 채팅
- 정답 숫자 3자리 생성 (서버, 1~9 중복 없음)
- 입력 유효성 검사 (자릿수 / 문자 포함 / 중복)
- S / B / OUT 판정 로직
- 시도 횟수 관리 (PlayerState, 현재/최대 표시)
- 승리 / 무승부 판정 및 결과 공지 위젯
- 게임 리셋 및 자동 재시작

### 도전 기능
- 실시간 타이머 위젯 (남은 시간 표시)
- 서버 기준 시간 동기화 (모든 플레이어 동일 시간)
- 시간 0 시 입력 차단
- 턴 전환 시스템 (입력 또는 시간 종료 시 다음 플레이어로)
- 턴당 시도 횟수 차감

### 추가 구현
- **게임 상태머신(FSM)**: `Waiting / Playing / Ending` enum으로 게임 흐름 관리
- **입력 차단 2중 방어**: 클라이언트 UI 잠금(편의) + 서버 측 차단(신뢰)
- **턴 / 시도 횟수 UI 표시**: "당신의 턴입니다", "시도: n/3" 화면 표시
- **이유별 유효성 안내**: 자릿수 / 문자 / 중복 등 원인별 메시지 구분

## 클래스 구조

| 클래스 | 역할 |
|---|---|
| GameModeBase | 정답 생성, 판정, 턴 제어, 승패 판정 (서버 전용) |
| GameStateBase | 게임 상태, 남은 시간, 현재 턴 (복제) |
| PlayerState | 플레이어별 이름, 시도 횟수 (복제) |
| PlayerController | 입력 처리, 위젯 소유, RPC |
| UI 위젯 | 채팅 입력/출력, 타이머, 결과 공지 |

## 실행 방법

1. `.uproject` 우클릭 → **Generate Visual Studio project files**
2. Visual Studio에서 빌드
3. 에디터에서 **Number of Players 2 이상**으로 플레이

## 주요 학습 내용

- 네트워크 복제(Replication)와 `DOREPLIFETIME`, `OnRep`
- 서버-클라이언트 책임 분리 (값 변경은 서버, UI 반응은 클라)
- RPC(Server / Client)의 실행 위치 이해
- 타이머 기반 상태 관리
